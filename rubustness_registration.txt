@startuml
actor GamePlayer

entity database

boundary "signing in screen" as signing_in_screen

boundary "Registration screen" as registration_screen 

control "Fill in userdata" as fill_in_userdata
control "Registration letter send" as registration_letter_sending
control "error display " as error_display #red
control "Registration letter confirm" as confirm #green

GamePlayer-- registration_screen 

registration_screen --> fill_in_userdata
fill_in_userdata --> error_display: "Unsuccessful"


fill_in_userdata --> registration_letter_sending : "Successful"

fill_in_userdata --> database: "Check"

registration_letter_sending --> signing_in_screen 

confirm --> database: "Changes" 

note as scenario 
Название прецедента: registration
Действующее лицо: game player
Цель: регистрация
Предусловия: 
game player запустил игру
нажал кнопку registration

Главаная последовательность:
1) game player вводит свой логин, пароль,
e-mail нажимает кнопку registration
2) система выполняет проверку введенных 
данных, таких данные в БД нет;
3) отправляется письмо на e-mail;
4) game player видит signing in screen.

Альтернативная последовательность 1:
1) game player вводит свой логин, пароль,
e-mail нажимает кнопку registration
2) система выполняет проверку введенных 
данных, такие данные в БД уже есть, следовательно
не заносит в БД данные;
3)  game player видит error screen.


Альтернативная последовательность 2:
1) Игрок поддетвердил полученное письмо регистрации
2) данные занесены в БД.

end note

@enduml
